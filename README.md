# RTB Rental Intelligence & Compliance Platform

## Project Objective

This project simulates a real-world data and analytics platform for a rental regulatory body.

The goal is to ingest rental, property registration, and case/dispute data, apply data quality and governance controls, build structured analytical models, expose trusted datasets, and deliver insights through Power BI reporting and API-ready data services.

## Business Problems Solved

1. Monitor rental price trends across counties and property types.
2. Detect missing, duplicated, or suspicious property registration records.
3. Track complaint/dispute volumes and resolution performance.
4. Provide clean datasets for statutory and management reporting.
5. Create a scalable Azure/Fabric-based architecture for analytics and integration.

## Project Phases

### Phase 1: Project setup and source data design
Create the source system structure and define realistic datasets.

### Phase 2: Raw data ingestion
Load rental, property, landlord, tenant, and dispute data into the raw layer.

### Phase 3: Data quality and cleansing
Apply validation rules, deduplication, missing value handling, and audit checks.

### Phase 4: Data modelling
Build clean star-schema models for rental analytics, compliance, and case reporting.

### Phase 5: Power BI reporting
Create dashboards for rental trends, compliance risks, and case performance.

### Phase 6: Integration and automation
Expose curated data through APIs and simulate workflow automation.
